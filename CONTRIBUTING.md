# Guide de Contribution

Merci de votre intérêt pour contribuer au projet VITALIS Backend !

## Standards de Code

### Style de Code

- Utiliser ESLint pour maintenir la cohérence
- Suivre les conventions JavaScript/Node.js
- Utiliser des noms de variables et fonctions descriptifs
- Commenter le code complexe

### Structure des Fichiers

- Un fichier par modèle/contrôleur/service
- Organiser les routes par domaine fonctionnel
- Garder les fonctions courtes et focalisées 

## Processus de Contribution  

1. **Fork** le repository
2. **Créer une branche** pour votre fonctionnalité (`git checkout -b feature/ma-fonctionnalite`)
3. **Commiter** vos changements (`git commit -m 'Ajout de ma fonctionnalité'`)
4. **Pousser** vers la branche (`git push origin feature/ma-fonctionnalite`)
5. **Ouvrir une Pull Request**

## Conventions de Nommage

- **Fichiers**: camelCase pour les fichiers JavaScript
- **Variables**: camelCase
- **Constantes**: UPPER_SNAKE_CASE
- **Classes**: PascalCase
- **Routes**: kebab-case dans les URLs

## Tests

- Écrire des tests pour les nouvelles fonctionnalités
- S'assurer que tous les tests passent avant de soumettre

## Documentation

- Mettre à jour la documentation si nécessaire
- Ajouter des commentaires JSDoc pour les fonctions publiques 

## Questions 

Pour toute question, ouvrir une issue sur le repository.
